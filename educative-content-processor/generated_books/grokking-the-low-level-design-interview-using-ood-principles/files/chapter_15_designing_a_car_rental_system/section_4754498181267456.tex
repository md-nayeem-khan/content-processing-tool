\subsection{Key requirements}\label{Iq7WPedOPmehRiTvv77-s}
\\
The following are the primary functional and operational requirements for the car rental system:

\begin{enumerate}
\item
\protect\phantomsection\label{QOzOm-08UQHqSkjTwoUNE}
Support two user roles: customer and receptionist.
\item
\protect\phantomsection\label{HVlyr9hIkBoaGFeaRA7IZ}
Manage multiple vehicle types: cars, trucks, vans, motorcycles (with subtypes).
\item
\protect\phantomsection\label{aNA1r4dG87pj5LvDqYHfP}
Track reservations, customer info, rental history, and vehicle logs.
\item
\protect\phantomsection\label{zPCdJYdvASkLPzxGVL6oZ}
Support the reservation life cycle: make, update, cancel, pick up, and return.
\item
\protect\phantomsection\label{7vWXd9oMRM4GMMX2NlNH2}
Allow optional services and equipment (e.g., Wi-Fi, child seat).
\item
\protect\phantomsection\label{Di-6Jbz3H9DHIoIWJ1irf}
Handle late returns and issue fines automatically.
\item
\protect\phantomsection\label{SytP-OlLWDXVc_wbjQs_d}
Enable vehicle search by type, model, features, or location.
\item
\protect\phantomsection\label{1wMsQb--ESyUj3tIMds-v}
Maintain multi-branch support with branch-specific inventories.
\item
\protect\phantomsection\label{rLE1fgfgq_I4vh4BbyO8u}
Track parking stall availability at each branch.
\item
\protect\phantomsection\label{PX9hsI6YBIyJ9M2uTvxmX}
Support secure payments and refunds via multiple methods.
\item
\protect\phantomsection\label{lr727NVwKS-9dXxEswQGg}
Allow payment at reservation time or vehicle return.
\end{enumerate}

\subsection{System actors}\label{OsOwlFNnNpZBy6xV6dliA}
\\
This system involves several actors with distinct roles and capabilities:

\begin{itemize}
\item
\protect\phantomsection\label{WC-aeyJ4rHlIMclPMesOs}
\textbf{Customer:} Registers, searches for vehicles, makes and updates reservations, selects services/equipment, makes payments, and handles vehicle pickup/return.
\item
\protect\phantomsection\label{9a0zKwHWxWNXEGTlG3R_4}
\textbf{Receptionist:} Performs customer operations on their behalf and manages vehicle inventory, logs, and payment processing at the branch.
\end{itemize}

\subsection{Design highlights}\label{cNrK6IXPpE82IdBZbPuns}
\\
In this section, we outline the design approach and the application of SOLID principles and design patterns to ensure modularity, flexibility, and scalability.

\subsubsection{Design approach}\label{KhQ8rbxsZ1fpzS-NJnHUB}
\\
The system follows a bottom-up approach, beginning with simple entities like \texttt{Vehicle}, \texttt{Customer}, and \texttt{Reservation}, and building up to high-level modules like inventory and payment management. It integrates subsystems to produce a cohesive, modular application.

\subsubsection{Design principles}\label{YsQlUTa-sxSCvy5W1p3-J}
\\
The system applies SOLID design principles as follows:

\begin{itemize}
\item
\protect\phantomsection\label{jYDvSks9PHOZQLl1d0xRY}
\textbf{Single Responsibility principle (SRP):} Each class (e.g., \texttt{Payment}, \texttt{Notification}, \texttt{Reservation}) encapsulates one well-defined responsibility.
\item
\protect\phantomsection\label{ljBscaPOT58tEZCVmM5wB}
\textbf{Open/Closed principle (OCP):} Services, equipment, and decorators can be extended without altering core logic.
\item
\protect\phantomsection\label{DWzDL4pfTazVuJsIYtsLW}
\textbf{Liskov Substitution principle (LSP):} Subclasses (e.g., \texttt{Car}, \texttt{Truck}) can be used wherever their base \texttt{Vehicle} type is expected.
\item
\protect\phantomsection\label{-UxBdzyOb18madbbCkMlp}
\textbf{Interface Segregation principle (ISP):} Specific interfaces like \texttt{Search} prevent dependency on unused operations.
\item
\protect\phantomsection\label{tuk67ehXAbc5PR8XPLJrj}
\textbf{Dependency Inversion principle (DIP):} High-level modules like \texttt{CarRentalSystem} depend on abstractions (\texttt{Search}, \texttt{Payment}, etc.), not implementations.
\end{itemize}

\subsubsection{Design patterns}\label{7-NrEzAq3q9yyNF5pJ9nD}
\\
Design patterns help structure the system's complexity:

\begin{itemize}
\item
\protect\phantomsection\label{MSxHN-j_VKu2R-3llG8YU}
\textbf{Decorator pattern:} Used for dynamic fee calculation (e.g., \texttt{DiscountDecorator}, \texttt{DamageFineDecorator}) without modifying core logic.
\item
\protect\phantomsection\label{sfdrIUPeQEXxqurBmuQ1z}
\textbf{Interface pattern:} Employed via the \texttt{Search} interface to allow flexible querying.
\end{itemize}

\subsection{Object interactions}\label{_Aq3DiKyVdfzIFtOXh_JV}
\\
Here, we describe the key message exchanges between objects that model real-time behaviors within the system.

\begin{itemize}
\item
\protect\phantomsection\label{w8ORudkoW5jlxWitvcBuJ}
\textbf{Vehicle reservation:}

\begin{itemize}
\item
Customer searches for vehicles via \texttt{VehicleCatalog}.
\item
Available vehicles are returned from \texttt{CarRentalSystem}.
\item
Customer selects a vehicle; system checks availability and creates a \texttt{VehicleReservation}.
\item
The system calculates fees and prompts for payment.
\item
Upon success, the reservation is confirmed, and a \texttt{Notification} is sent.
\end{itemize}
\item
\protect\phantomsection\label{A0l1zSNycNmT5B0jo1t1a}
\textbf{Reservation cancellation:}

\begin{itemize}
\item
Customer requests cancellation.
\item
System verifies policy and updates reservation status.
\item
Any applicable refund or fine is processed via \texttt{Payment}.
\item
A cancellation \texttt{Notification} is sent.
\end{itemize}
\item
\protect\phantomsection\label{hFdWa_nRz7kSaTW7NBmR6}
\textbf{Overdue return:}

\begin{itemize}
\item
Upon late return, the system detects overdue via the \texttt{Reservation}.
\item
A \texttt{Fine} is applied.
\item
A \texttt{Notification} alerts the customer about the overdue charge.
\end{itemize}
\item
\protect\phantomsection\label{91tggfww5lFocytkwnl7f}
\textbf{Add services/equipment:}

\begin{itemize}
\item
During or after the reservation, the customer selects optional items.
\item
\texttt{Service} or \texttt{Equipment} objects are added to the reservation.
\item
Total cost is recalculated.
\end{itemize}
\end{itemize}

\subsection{System workflow}\label{FZUizPbyEkD4ImZ08FTMN}
\\
This section outlines the main workflows that occur in the system, as illustrated in the activity diagrams.

\begin{itemize}
\item
\protect\phantomsection\label{6iJdCFlpIYvaGGnAgc77-}
\textbf{Vehicle reservation:}

\begin{itemize}
\item
Customer searches vehicle inventory by date, location, or model.
\item
Selects and reserves a vehicle with optional services/equipment.
\item
Makes payment during the reservation or upon return.
\end{itemize}
\item
\protect\phantomsection\label{WAO-pGhtIiFHtO1Vjs6lC}
\textbf{Vehicle pickup:}

\begin{itemize}
\item
The customer arrives at the branch.
\item
The receptionist verifies the reservation and updates the vehicle status.
\item
The vehicle is handed over to the customer.
\end{itemize}
\item
\protect\phantomsection\label{0Xd7somWh1zzFgLPfXwX8}
\textbf{Vehicle return:}

\begin{itemize}
\item
Vehicle is returned at the designated branch.
\item
The receptionist inspects for damage or refueling.
\item
Applicable fines are assessed.
\item
Final payment is collected, and vehicle status is updated.
\end{itemize}
\item
\protect\phantomsection\label{UzKB-wNXkKz0t7W9cNWx-}
\textbf{Optional services handling:}

\begin{itemize}
\item
During reservation updates, the customer selects add-ons.
\item
Services like \texttt{Wi-Fi} or Roadside\ Assistance are added to the reservation object.
\item
Payment is adjusted accordingly.
\end{itemize}
\item
\protect\phantomsection\label{7smvXn4M7_mTWmhloLtMI}
\textbf{Notifications workflow:}

\begin{itemize}
\item
System sends email/SMS for confirmations, cancellations, and overdue alerts.
\end{itemize}
\end{itemize}
