\subsection{Design highlights}\label{XPAE1pHZBHZ9Br522mZVR}

This section summarizes key design decisions and rationale in Facebook's LLD:

\subsection{Design approach}\label{M7iG-D1jXA1sz0ERog4MO}
\\
The Facebook system uses a \textbf{bottom-up approach}, starting with fundamental units (posts, comments), then composing them into more complex features (profiles, groups, pages), and integrating them into the full platform.

\subsection{Design principles}\label{nFcGApokayx9dIDZo_E0I}
\\
The design demonstrates the SOLID principles:

\begin{itemize}
\item
\protect\phantomsection\label{GI8lO4xLeU0e6e1rMdvyx}
\textbf{Single Responsibility principle (SRP):} Each class handles one clear aspect (e.g., \texttt{Profile} for bio info, \texttt{Notification} for alerts).
\item
\protect\phantomsection\label{BEUuEG4xeSbxTSWYri4p4}
\textbf{Open/Closed principle (OCP):} By extending classes/interfaces, new features (e.g., new post/privacy types) can be easily added.
\item
\protect\phantomsection\label{9k0Zxzzz4TA76_HlSeghL}
\textbf{Liskov Substitution principle (LSP):} Subclasses (\texttt{User}, \texttt{SearchCatalog}, \texttt{Group}) can be used wherever their parent/interface is expected.
\item
\protect\phantomsection\label{l9upWJaS8IWt_ABYiVD_Q}
\textbf{Interface Segregation principle (ISP):} Distinct interfaces for searching, group/page/post/comment functions keep code modular.
\item
\protect\phantomsection\label{FG_6YVYgbVyJJKOwRx2H3}
\textbf{Dependency Inversion principle (DIP):} High-level features depend on abstractions (e.g., \texttt{Search}, privacy settings) instead of concrete implementations.
\end{itemize}

\subsection{Design patterns}\label{zWOM1GFToLaZKPJx-4B-2}
\\
Key patterns in Facebook's system include:

\begin{itemize}
\item
\protect\phantomsection\label{Gi9x2niLuea9GWAGHwZ8W}
\textbf{Observer pattern:} Used for notifications; users are notified of events like messages, friend requests, and post activity.
\item
\protect\phantomsection\label{3ORewlh6JbsZJ1ky5qzWR}
\textbf{Singleton pattern:} Ensures the notification system has a single control point for sending user alerts.
\item
\protect\phantomsection\label{2X2kPdecQWyJkU24TIdmg}
\textbf{Factory pattern:} Used to create instances of \texttt{User}, \texttt{Post}, and \texttt{Page} objects, which allows for flexible object creation and management.
\item
\protect\phantomsection\label{rzd24xh6R5xWTxzZSsrK7}
\textbf{Strategy pattern:} This pattern enables the dynamic selection of privacy settings or feed ranking strategies for posts and profiles, allowing users and the system to adapt visibility and personalization features flexibly.
\item
\protect\phantomsection\label{OfeZUWUxqP0Bgiap2kdSo}
\textbf{Interface Segregation and Abstraction:} The system uses distinct interfaces (such as \texttt{Search}, \texttt{PageFunctionsByUser}, \texttt{GroupFunctionsByUser}, \texttt{PostFunctionsByUser}, and \texttt{CommentFunctionsByUser}) to separate different user and admin actions. This keeps the code modular, focused, and easy to extend with new features.
\end{itemize}

\subsection{System workflow}\label{B4E9s4q3UBZaDVVNZVRL7}
\\
This section outlines the main end-to-end scenarios for Facebook users:

\begin{itemize}
\item
\protect\phantomsection\label{AuV0Co4XpK-grkB9NK3WZ}
\textbf{User registration and profile setup:} A new user enters personal details like name, address, phone number, and email, then adds work experience, education, and places lived. Users can configure profile privacy settings to control who can view their information.
\item
\protect\phantomsection\label{9qre6dlYIEvBIbtEUEoGL}
\textbf{Content discovery and engagement:} Users use the search feature to find other users, groups, pages, or posts on the platform. They can also follow users, join groups, or follow pages and interact by liking, commenting, or sharing posts and comments.
\item
\protect\phantomsection\label{NWiwMtqBARIJdj6vDp9ir}
\textbf{Building connections:} Users connect with others by sending friend requests, which can be accepted or rejected. They can also follow users without sending a friend request to see their public activities, and unfriend or block others if needed.
\item
\protect\phantomsection\label{QgCOCA4_3aWBgDtodYP7G}
\textbf{Content creation and sharing:} Users create posts and set privacy preferences for each one. Shared posts appear in the feeds of friends or followers, who can then like, comment on, or share them.
\item
\protect\phantomsection\label{LP-GXeBTSWz9BbZA17A5Q}
\textbf{Group/page interactions:} Users can create groups or pages, join or follow existing ones, and set privacy options. Groups and pages enable users to connect with communities and discuss shared interests.
\item
\protect\phantomsection\label{ZE_fn84dJ6xQd9_H-dST9}
\textbf{Notifications and messaging:} The system automatically sends notifications for friend requests, messages, new posts, and comments. Users can send and receive direct messages to communicate privately with other users.
\item
\protect\phantomsection\label{NAVaM1lKC6NByeyIYWJot}
\textbf{Privacy and security:} Users manage privacy settings for their profiles, posts, and groups, deciding who can view or interact with their content. The system also allows users to block or unblock other users when needed.
\end{itemize}
