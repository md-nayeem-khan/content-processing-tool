\subsection{Design approach}\label{DbGHNucVFx-IgRCZxbKaC}
\\
The system uses a bottom-up approach, beginning with atomic units (table, seat, menu item) and building up to meals, orders, reservations, and finally, full branch and restaurant management.

\subsection{Design principles}\label{xob1VB7QMYimXbMw-6hIq}
\\
The design demonstrates the following SOLID principles:

\begin{itemize}
\item
\protect\phantomsection\label{9SEI_2_EV68YI9eneFB5F}
\textbf{Single Responsibility principle (SRP):} Each class has a focused purpose (e.g., \texttt{Waiter} manages orders, \texttt{Notification} handles alerts).
\item
\protect\phantomsection\label{K3VQWbfjpf_oy1sc0U1vj}
\textbf{Open/Closed principle (OCP):} Extending base classes allows adding new payment types, menu sections, or notification methods.
\item
\protect\phantomsection\label{XrRinhA5307smvcnwY7o-}
\textbf{Liskov Substitution principle (LSP):} Derived classes (e.g., \texttt{Customer}, \texttt{Waiter}) can replace their parents without affecting system behavior.
\item
\protect\phantomsection\label{oi9a8hLMTaT_GJFKEdBhF}
\textbf{Interface Segregation principle (ISP):} Functionalities like reservations, ordering, and payments are handled by separate classes, avoiding ``fat'' interfaces.
\item
\protect\phantomsection\label{uB1vkUd4XHgrisvIWqK_t}
\textbf{Dependency Inversion principle (DIP):} High-level modules depend on abstractions (\texttt{Payment}, \texttt{Notification}) rather than concrete implementations.
\end{itemize}

\subsection{Design patterns}\label{1_p53MM2SEKL6SbwJx8kg}
\\
Several established patterns are used:

\begin{itemize}
\item
\protect\phantomsection\label{NsrxgkptiRCmcb15btZqF}
\textbf{Strategy pattern:} This pattern is used for payment processing. The system supports different payment strategies (cash, credit card), allowing the payment process to be dynamically selected and extended.
\item
\protect\phantomsection\label{X4JpgmX1NSTjReVZDT3Rh}
\textbf{Template Method pattern:} Each payment type (cash or credit card) follows the same overall process (prepare bill, initiate transaction, finalize payment), but can implement steps differently. This common flow with customizable steps is modeled with the template method pattern.
\item
\protect\phantomsection\label{ssNNM46Ih5FmPuVu9IXOm}
\textbf{Composite pattern:} The restaurant menu is structured hierarchically: a menu contains multiple sections (like starters or desserts), each containing multiple items. This whole-part relationship is modeled using the composite pattern.
\end{itemize}

\subsection{System workflow}\label{JVfdWAil_fvAYNgBdnLZD}
\\
This section describes the main user journeys and life cycle events:

\begin{itemize}
\item
\protect\phantomsection\label{WxN8iiDr-v3aGOKnb_67X}
\textbf{Branch setup and management:} The manager configures each branch's table layouts and updates the menu and table configurations as required. This ensures every branch has an accurate seating plan and an up-to-date menu for customers and staff.
\item
\protect\phantomsection\label{-xWh-VGdEsSm3Ghte-tRp}
\textbf{Reservation flow:} Customers or receptionists search for available tables by date and time to make, modify, or cancel reservations. As reservation times approach, the system automatically sends notifications to customers and updates table statuses accordingly.
\item
\protect\phantomsection\label{RicH1Y69t3jAxaAEhoeAf}
\textbf{Walk-in ordering and dining:} Customers request tables at the reception and are seated if available. The server provides the menu, takes orders, and assists the customer throughout their dining experience.
\item
\protect\phantomsection\label{wgrEGLyqe9bfFmAR21fNw}
\textbf{Order and billing process:} The waiter creates or updates orders for the table, adds meal items for each customer, and generates the bill after service. The bill reflects all items ordered and includes any applicable taxes.
\item
\protect\phantomsection\label{IKyEfLqlCIiv5252NaQT3}
\textbf{Payment handling:} Customers pay the bill by cash or credit card, which the waiter processes after the bill is generated. The system records the payment and marks the table as available for future reservations or walk-ins.
\item
\protect\phantomsection\label{1D43D4BL30SKp96T-Hpg3}
\textbf{System-driven updates:} The system sends automatic notifications to customers as their reservation time approaches and updates table statuses after reservation and payment events. This keeps all actors informed and the restaurant's table management accurate in real time.
\end{itemize}
