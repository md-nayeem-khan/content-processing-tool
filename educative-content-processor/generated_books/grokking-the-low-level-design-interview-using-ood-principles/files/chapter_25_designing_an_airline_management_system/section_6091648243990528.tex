\subsection{Design approach}\label{-tHpAIDT6Ch1Ya3M-KEEe}
\\
The system employs a bottom-up approach, starting with the smallest building blocks, such as seats and flight reservations, which are then composed to create larger, more complex components, including aircraft, flights, itineraries, and the entire management system.

\subsection{Design principles}\label{CP5ikBc2Tx4BtKbJkbz7g}
\\
The design follows key SOLID principles:

\begin{itemize}
\item
\protect\phantomsection\label{lHsQj2vVwaBIajVbm0Gt5}
\textbf{Single Responsibility principle (SRP):} Each class is focused on one aspect, e.g., \texttt{FlightSeat} for seat allocation, \texttt{Payment} for transaction processing, \texttt{Notification} for alerts.
\item
\protect\phantomsection\label{fbejhDJ4qkCfFC0EA0l-2}
\textbf{Open/Closed principle (OCP):} The system allows adding new payment types, seat classes, or notification mechanisms without altering existing code.
\item
\protect\phantomsection\label{3WClwEXdqSiyIjweTqanu}
\textbf{Liskov Substitution principle (LSP):} Subtypes like \texttt{Admin}, \texttt{Crew}, \texttt{FrontDeskOfficer}, and \texttt{Customer} can substitute their base \texttt{Person} class.
\item
\protect\phantomsection\label{wiPddJwO8ZGe7-XBOqaWt}
\textbf{Interface Segregation principle (ISP):} Search, payment, and notification features are separated into distinct interfaces and implementations.
\item
\protect\phantomsection\label{znhO2TSzsJcU0IVd5qoTu}
\textbf{Dependency Inversion principle (DIP):} High-level modules depend on abstractions (e.g., \texttt{Payment}, \texttt{Notification}) rather than specific implementations.
\end{itemize}

\subsection{Design patterns}\label{KWFaHQgjLmzx2_Cbe3fVS}
\\
The system leverages several established design patterns:

\begin{itemize}
\item
\protect\phantomsection\label{HJW8K5zh-efYgdy3hn1mk}
\textbf{Singleton pattern:} Ensures only one \texttt{Airline} instance is available throughout the application.
\item
\protect\phantomsection\label{DyadHm2Arg2rIZwIX2M_Q}
\textbf{Observer pattern:} Enables passengers to be automatically notified of any changes to their flight reservations or status.
\end{itemize}
