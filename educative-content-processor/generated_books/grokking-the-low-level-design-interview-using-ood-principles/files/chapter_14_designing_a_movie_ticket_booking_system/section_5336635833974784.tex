The movie ticket booking system is designed to facilitate searching for movies, viewing showtimes, and booking tickets across multiple cinemas and their halls. The system allows online customers and in-person ticket agents to manage bookings and payments, enabling administrators to control movie and show listings. It incorporates real-time seat selection, various payment methods, and an automated notification system for relevant events.

\subsection{Key requirements}\label{v_dDDXgKJIvX60eC_tFhi}
\\
The following are the key requirements of the case study:

\begin{enumerate}
\item
\protect\phantomsection\label{7bgZUB8iOsJUqhX29iQMj}
The system must support multiple cinemas in different cities, each with multiple halls, where each movie can have various shows, but a hall hosts only one show at a time.
\item
\protect\phantomsection\label{HUV57zh7hbjb2YcYNbRzF}
The system must display all available showtimes for selected movies across all cinemas.
\item
\protect\phantomsection\label{YuS1NE1yxQhEVT_qn-jvP}
Users can search for movies by title, language, genre, or release date.
\item
\protect\phantomsection\label{Y8sjUhC_GCrP8U_yu92v9}
Users can book multiple available seats for any showtime in a single transaction, online or in person via a ticket agent.
\item
\protect\phantomsection\label{ulRDANPtPL1kgRGdhAqiR}
Seats are categorized into ``Silver,'' ``Gold,'' and ``Platinum'' with fixed costs. Only one ticket can be allocated per seat, and the system prevents duplicate bookings for the same seat and show.
\item
\protect\phantomsection\label{n80Rczt-RxijxLVB2p80e}
Online customers can pay by credit card only, while in-person customers can pay by cash or credit card through a ticket agent.
\item
\protect\phantomsection\label{C0EVOa--OsJN8Y5IMdeJY}
The system must visually distinguish between available and booked seats.
\item
\protect\phantomsection\label{TIIDxznRNouIkkd9YPhax}
Admin users must be able to add, delete, or update shows and movies.
\item
\protect\phantomsection\label{PzBTicLOThEICIod09-kF}
The system must generate notifications for new movie releases, booking confirmations, cancellations, and booking modifications.
\end{enumerate}

\subsection{System actors}\label{lJ92a1xM1DB1BY8-OnHlR}
\\
The following are the main actors in the case study:

\begin{itemize}
\item
\protect\phantomsection\label{TOABnXFGHcxLz1L-wvN_h}
\textbf{Customer:} Searches for movies, creates bookings, reserves seats, pays for tickets, and views/modifies/cancels bookings.
\item
\protect\phantomsection\label{H_6xOrYFFZu4JqP5oqZEZ}
\textbf{Ticket agent:} This person assists customers by creating bookings, searching for movies, reserving seats on their behalf, and accepting payments, but they cannot modify or cancel a booking.
\item
\protect\phantomsection\label{mhmRCIfO2_3Tv50z6HKnu}
\textbf{Admin:} Manages movies and showtimes by adding, deleting, or updating them.
\end{itemize}

\subsection{Design highlights}\label{WuAtP6dLnkKTvSMUnHb_x}
\\
This section outlines the key design methodologies and principles in designing a movie ticket booking System.

\subsubsection{Design approach}\label{tENzcFZ_sjYwyO2-CP2Pw}
\\
The system follows a bottom-up design methodology, from defining entities like seats, halls, and users to integrating them into a cohesive system structure.

\subsubsection{Design principles}\label{oKiL9B_rk8P9oR_wBg-FR}
\\
The following are the design principles that are applied to the design of the case study:

\begin{itemize}
\item
\protect\phantomsection\label{jGzR0SBylT6O9fMuQh0HC}
\textbf{Single Responsibility principle (SRP):} Each class handles a distinct responsibility, e.g., \texttt{Seat} handles seat info, \texttt{Booking} manages booking logic.
\item
\protect\phantomsection\label{fMx9JyICv3iupsY7MspFH}
\textbf{Open/Closed principle (OCP):} The \texttt{Seat} class allows for extension (Silver, Gold, Platinum) without modification.
\item
\protect\phantomsection\label{Xy8vDp3TiZ4VKtlmwSbYH}
\textbf{Liskov Substitution principle (LSP):} Subclasses like \texttt{CreditCard} and \texttt{Cash} can substitute \texttt{Payment} without affecting the system.
\item
\protect\phantomsection\label{SZ5ylfnTi23RQxdAt2NVu}
\textbf{Interface Segregation principle (ISP):} The \texttt{Search} interface defines focused methods for searching movies without coupling to unrelated functionalities.
\item
\protect\phantomsection\label{KrkQAqYHnYPtS6cyQfHqW}
\textbf{Dependency Inversion principle (DIP):} High-level classes like \texttt{Catalog} depend on abstractions (\texttt{Search} interface), not concrete implementations.
\end{itemize}

\subsubsection{Design patterns}\label{mZFrQ-AbUDI8wDa72LN0t}
\\
The following are the design patterns that are applied to the design of the case study:

\begin{itemize}
\item
\protect\phantomsection\label{0tFHR3Didtokdr-9lm6v8}
\textbf{Factory pattern:} This pattern creates various \texttt{Seat} types based on category (Silver, Gold, Platinum).
\item
\protect\phantomsection\label{f_5dduth8_QK2av_HVagq}
\textbf{Observer pattern:} The notifications system uses observers to send updates on movie releases, bookings, and changes.
\item
\protect\phantomsection\label{YM6qeg5MdBnYlRn7TJ-to}
\textbf{Strategy pattern:} The \texttt{Search} interface allows interchangeable search strategies (by title, genre, language, etc.).
\item
\protect\phantomsection\label{5UZ2gfHuBIuf0Gjc1s9FD}
\textbf{Singleton pattern:} This pattern ensures that central classes like \texttt{Catalog} or the notification dispatcher have only one global instance managing state.
\item
\protect\phantomsection\label{NY-Ly8KlIkG6kSkxJQ9_Z}
\textbf{Builder pattern:} Facilitates the step-by-step construction of complex \texttt{Booking} objects, especially involving multiple seat selections.
\end{itemize}

\subsection{Object interactions}\label{Eca7kOiWLIEqIFcnUzPQN}
\\
The following are the main object interactions:

\begin{itemize}
\item
\protect\phantomsection\label{GCpiAiIfsmPX4oo-A5yG7}
\textbf{Ticket issuance:} The customer or ticket agent selects a movie and showtime, reserves seat(s), confirms booking, and receives the ticket.
\item
\protect\phantomsection\label{UqUFjPN4I0uojVsxRZpfB}
\textbf{Fee calculation:} The system computes the total cost and handles payment based on seat category (Silver/Gold/Platinum).
\item
\protect\phantomsection\label{KCM0zVb7muc0yNNNst_sK}
\textbf{Notification generation:} Upon booking, cancellation, or movie release, the system dispatches relevant notifications to users.
\item
\protect\phantomsection\label{50U1n7Nfqfd1RRCu91jpN}
\textbf{Booking modification:} When the customer modifies the booking, the system updates the seat map and notifies the user.
\item
\protect\phantomsection\label{Y8ZmmTBx9dbZmTkJEz_-i}
\textbf{Cancellation:} The booking is canceled, the seat status is reverted, a refund (if applicable) is initiated, and a cancellation notice is sent.
\end{itemize}

\subsection{System workflow}\label{d9WokN-h28v0YUL9_ChOy}
\\
The following are the main workflows that we have worked on in the case study:

\begin{itemize}
\item
\protect\phantomsection\label{McUDH5nGqEpsdnexZL4Ok}
\textbf{Customer booking flow:}

\begin{itemize}
\item
Search for a movie using filters (title, genre, language, date).
\item
Select a showtime from available listings.
\item
Choose seat(s) on the seating map.
\item
Make a payment via credit card.
\item
Receive booking confirmation and ticket.
\end{itemize}
\item
\protect\phantomsection\label{IFJOCUaWMw0iujEQYo6pf}
\textbf{Admin management flow:}

\begin{itemize}
\item
Add or update a movie and its shows.
\item
Modify or delete existing shows (which may trigger notifications).
\item
Remove a movie entirely, canceling all associated shows.
\end{itemize}
\item
\protect\phantomsection\label{OQs-ISBynbbUeTOMoUkHv}
\textbf{Ticket agent flow:}

\begin{itemize}
\item
Perform movie searches and create bookings for walk-in customers.
\item
Accept payment in cash or by credit card.
\item
View and manage customer bookings (no modification/cancellation rights).
\end{itemize}
\item
\protect\phantomsection\label{gZTM0BYUcoYzDFJIDZq3m}
\textbf{System notification flow:}

\begin{itemize}
\item
Automatically send booking confirmation, modification, or cancellation alerts.
\item
Notify opted-in users about new movie releases.
\end{itemize}
\end{itemize}
